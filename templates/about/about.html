About DrinksAPI

What's an API?

Wikipedia defines an application programming interface (API) as a way for two or more computer programs or components to communicate with each other. It is a type of software interface, offering a service to other pieces of software.[1] A document or standard that describes how to build or use such a connection or interface is called an API specification. A computer system that meets this standard is said to implement or expose an API. The term API may refer either to the specification or to the implementation. Whereas a system's user interface dictates how its end-users interact with the system in question, its API dictates how to write code that takes advantage of that system's capabilities.

What's the DrinksAPI about?

It's a project that started with a few entries to better understand API development. Since retreiving and manipulating API data is a large part of web development, I've made the DrinksAPI available to anyone that want's another data set to practice on.

The "must-know" endpoint for the DrinksAPI returns cocktail recipes that were deemed very popular among drinkers according to the all knowing internet.

What data is in the API?

  • name of cocktail or shot | string
  • slug of cocktail or shot name | string
  • flavor profile of the cocktail | string
  • base alcohol needed for the drink | list
  • ingredient names | list
  • garnish | list
  • serving glass | string
  • mixing directions | string
  • drink type | string
  • must know drink | boolean

name of cocktail or shot

"drink_name": "Tom Collins"

Provides name of the drink

slug of cocktail or shot name

"slug": "tom-collins"

Provides a slug for use in a website url.

flavor profile of the cocktail

"profile": "Refreshing"

Provides a flavor profile for the drink or shot. I've updated the profiles for the drinks I know. For other's I used the all knowing, always correct, information super highway to get the flavor profiles. For those drinks that I haven't tried or couldn't find a simple flavor profile for, I left as "N/A".

base alcohol needed for the drink

"base_alcohol": "Gin"

Provides the liquor or liqueur that serves as the main ingredient for the cocktail or shot.

ingredient names

"ingredient_names": ["2.00 oz Club Soda", "1.00 oz Lemon Juice", "2.00 oz London Dry Gin", "0.50 oz Simple Syrup"]

Provides the ingredients needed to make the cocktail or shot. Warning: If you're a cocktail enthusiast, who takes issue with a recipe not being exactly how you know it be, chill out. These recipes are a guide for creating a decent drink, they are not the de facto instructions on concocting cocktails. So if you add crispy bacon to your Bloody Mary or you hold the grenadine on your Tequila Sunrise, great, do you!

garnish

"garnish": ["Lemon Wheel", "Marachino Cherry"]

Provides the recommend garnish for the cocktail or shot. Warning: To the cocktail aficionados, these are only recommondations. If the API says garnish with a lime wedge and you prefer to do a lemon slice, do you!

serving glass

"drink_name": "Collins/Highball"

Provides the recommended glass to serve the cocktail in. Martinis shouldn't be served in old-fashioned glasses and Old Fashions shouldn't be served in hurricane glasses, but red Solo cups are acceptable for all cocktails and shots!

mixing directions

"mixing_directions": "1. Add lemon juice, simple syrup, and gin to glass\r\n2. Add ice to glass.\r\n3. Top off with club soda.\r\n4. Stir drink.\r\n5. Garnish with lemon wheel and maraschino cherry"

Provides instructions on how to build the cocktail or shot.

drink type

"drink_type": "cocktail"

Seperates drinks into two categories, cocktail or shot.

must know drink

"must_know_drink": "true"

Must know drinks are cocktails or shots that I think aspiring bartenders, party hosts and home mixologists should know. Adding this was inspired by me - the patron - having to tell a bartender how to make a gin & tonic.

Why was the DrinksAPI created?

Practice! The DrinksAPI was a project started in order to gain experience working with the Django Rest Framework. The project provided the challenge of creating a simple API, which I can use to practice fetching and displaying API data with frontend frameworks like React.js or Next.js.

What's the technology stack?

  • Javascript
